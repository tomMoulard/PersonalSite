Question Three : Describe the ANSI-SPARC three-level architecture of DBS, what are the advantages of this architecture ?

This architecture is divided in three levels :
The External Level, It allow users to view a part of the database that is relevant to him, and forbids the user to see an unauthorized file.
The Conceptual Level, a useful level to see the whole database without constraints and show the inter-connection of data. But this level does not explain how the data is stored physically.
And then The Internal Level, represent how the physic database is managed.
There is a few advantages to use a ANSI-SPARC three-level architecture, like different user view depending of the user, and the user does not have to know how the data is managed. Moreover, the DBAdmin can change storage structures without affecting the user experience.

Question Four : What is Data Independence(DI) ?

The Data Independence is a way to express data in a level of abstraction that allows data in others level to be changed without being a constraint for this data. You can therefore change the structure of the database without affecting a good usage for a program or a user.

These two level of DI can change both level of schema, the logical schema for the LDI and physical schema for the PDI. Except that it only changes its own level.

Part Three

Describe CAP(Consistency, Availability, Partition-tolerance)/BASE(Basically Available, Soft-state, Eventually consistent)/ACID

CAP

The CAP Theorem is composed of three main guaranties Consistency, Availability and Partition-Tolerance. Each one of them is possible to archive but the computer scientist Eric Brewer stated that it is impossible to provide a computer system with each three of these guarantees. The max possible inside a system is two out of three.
Consistency ensure that every read receives the most recent write or an error.
Availability require to have a full time functional system that give a response for each request.
Partition-Tolerance explain that even if there is a subnet failure, this system is still usable.

BASE

BASE is a label to classify consistent services to contrast with a more traditional ACID. In fact, this label promote availability over consistency. Data operations run with less efficiency without user's notice.

ACID

This is a set of elemental transactions made inside the database : Atomicity, Consistency, Isolation, Durability. Each one describe the fundamentals features of a DBMS.
Atomicity require that transactions should be accomplished, and in the case there is something wrong in the procedure, the entire transaction is aborted.
Consistency is when one transaction is accomplished the database is still functional.
Isolation explain that each transaction should not have any interference with an other transaction even if there is another undergoing transaction.
Durability ensure that when a transaction is accomplished successfully even a power loss cannot erase it.
